  • What a responsive site is, and why responsiveness is important.

    A site the alters its layout depending on the size of the browser window/screen. It's useful so that a screen of any kind can view the website without elements being the wrong size and therefore hindering usability.

  • What mobile first design is, and why it's important.

    Mobile first design is creating the website first to look good on smaller screens, and then scaling it up to also look good on larger screen. This is important because it is easier to shuffle good looking elements to fit a larger grid layout than it is to condense a grid layout into a single column view.

  • What frameworks are, and their pros and cons.

    A CSS framework is a ready-built CSS document that should use good standards and should have many useful tools already implemented. This takes a lot of the work out of creating a new webpage, and helps in making a good webpage. The downside of this is that it requires reading of the framework's documentation (and it may take more effort to actually understand how to use the framework after that).

  • What a wireframe is and why we use it.

    A wireframe is a plan for a webpage that shows how elements should be laid out. It makes it much easier to proportion elements of the webpage, and makes it easier to understand the layout.
