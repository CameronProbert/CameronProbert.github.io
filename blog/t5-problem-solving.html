A simple problem I got blocked on

A simple problem I got stuck on was when making a heuristic for a roadmap in Java. I needed to find a function that would represent the fastest way from A to B. The answer was the time taken to travel the direct distance (as the crow flies, not by road), by going at the max speed limit (100km/hr in New Zealand).

While trying to solve this problem I talked to my friends who were also tring to figure it out, and we used trial and error to test some different heuristics.
I was annoyed that I couldnt figure out how to solve this problem, as it was necessary to get an A for the university assignment.
I learned that I can't always solve a problem, but other people did figure the answer out, so I can always ask others for help if I need it.

A problem I solved elegantly

At university I had an assignment to make a webpage using Javascript that would be used on wearable glasses (such as Google Glasses). These glasses would have swipe controls on one side of the glasses, giving 5 inputs. Swipe forward, backwards, up, down and a tap. I had to make an interface that would work well with these glasses, but the problem was testing it using the swipe controls. I ended up implementing 5 buttons that would simulate each of these inputs.


I think this worked well. It wouldn't be able to gauge the speed of the swipes, but for my program that didn't matter. What I liked about this solution is that it was so simple compared to the much more complicated things I was doing for the rest of the assignment.
Sometimes simple solutions can be the most elegant.

Googling

If I am trying to solve a problem that I do not know the answer to, I will usually use Google to search for peoples' answers to similar problems.

Asking your peers for help

If Google doesn't help me, or I can see that a peer has already solved the problem, then I will ask a peer for help. This usually leads to more actual learning than Googling a problem, as well as better relations with the peer.

Asking coaches for help

If I still don't have an answer I will then ask a coach for help. I think it is usually better to ask a peer first though, unless in a case where I want to know best practice or alternate solutions to something I've already found.
